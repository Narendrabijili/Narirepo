🔹 What is Namespace Resources?

Namespace resources = Namespace is a logical separation inside a Kubernetes cluster.

"In a Kubernetes cluster, we create logical isolated spaces called Namespaces. Namespaces are used to separate different environments such as dev, test, and prod, or different projects, while sharing the same cluster resources."
Syntax:

apiVersion: v1
kind: Namespace
metadata:
  name: expense
  labels:
    project: expense
    environment: dev

kubectl apply -f yaml.file

kubectl get namespaces





🔹Pod Resources: It is the smallest deployment in K8.In Pod we can have on or multiple containers.

apiVersion: v1
kind: pod
metadata:
  name: nginx
  labels:
    purpose: demo
spec:
  containers:
   - name: nginx
     image: nginx

kubectl apply -f yaml.file

kubectl get pods



Multi IMAGE creation:


apiVersion: v1
kind: pod
metadata:
  name: multi-ca-creation
  labels:
    purpose: demo
spec:
  containers:
   - name: nginx
     image: nginx

   - name: almalinux
     image: almalinux:9

Here we get famous error CrashLoopBackOff error it nothing but unable to start CA. there is 1/2 due to in almalinux CA there is only os there is no continuous starting processes not starting 

so we need to add command to run continuous only.

command: ["sleep","1000"] == this need to add to almalinux then both ca will run.



#watch kubectl get pods == to see what are running 


#kubectl describe pod multi-ca-creation== to error details 

We created two CA.to login to CA

#kubectl exec -it name -- bash ==Command for login to CA.



#kubectl exec -it multiple-ca-creation  -c almalinux -- bash ==Command for login to particular CA in case of multi ca creation scenario 


IF we created CA1 and CA2 in same pod there share same network and storage also. if we deleted pod then in pod then data(like logs) also will be delated .so data inside pod is temporary.
so we need to store logs in elasticsearch  server .then will be done by sidecar CA.
